====================================================
MODUL JALUR PENJUALAN
====================================================

Default Jalur:

1. WA
2. Marketplace
3. Web

Field:
- Nama Jalur
- Persentase Insentif
- Status Aktif

Contoh:
- WA = 3%
- Marketplace = 0.5%
- Web = 2%

Persentase bisa diubah oleh HRD kapan saja.

====================================================
MODUL INSENTIF SALES
====================================================

Konsep:
Insentif dihitung berdasarkan performance penjualan.

Rumus:
Insentif = Nominal Penjualan x Persentase Jalur

====================================================
JALUR WA
====================================================

HRD input manual transaksi penjualan WA.

Field:
- Tanggal
- Karyawan
- Customer
- Nominal Penjualan
- Keterangan

Sistem otomatis menghitung insentif berdasarkan persentase jalur WA.

====================================================
JALUR MARKETPLACE
====================================================

HRD input total penjualan marketplace per periode.

Field:
- Periode
- Cabang
- Total Penjualan Marketplace

Lalu HRD membagi porsi performance ke masing-masing karyawan.

Contoh:
Total Marketplace = 100.000.000

Pembagian:
- Karyawan A = 50%
- Karyawan B = 30%
- Karyawan C = 20%

VALIDATION:
- total pembagian wajib 100%
- tampilkan warning jika tidak 100%

Sistem otomatis menghitung:
Performance Karyawan =
Total Marketplace x Persentase Pembagian

Lalu hitung insentif:
Performance x Persentase Jalur Marketplace

====================================================
JALUR WEB
====================================================

Konsep sama seperti marketplace.

HRD input:
- periode
- total penjualan web
- pembagian %

VALIDATION:
- total pembagian wajib 100%

====================================================
MODUL INSENTIF AKTIVITAS
====================================================

Buat master aktivitas fleksibel.

Contoh:
1. Live Stream
2. Konten

Field:
- Nama Aktivitas
- Tipe Perhitungan
- Nominal Insentif
- Status Aktif

Tipe Perhitungan:
- per jam
- per qty

====================================================
INPUT AKTIVITAS KARYAWAN
====================================================

HRD dapat input aktivitas karyawan.

Contoh:
Live Stream:
- 10.000/jam

Karyawan A:
- 4 jam

Maka:
4 x 10.000 = 40.000

Konten:
- 15.000/konten

Karyawan:
- 10 konten

Maka:
10 x 15.000 = 150.000

Field:
- tanggal
- karyawan
- aktivitas
- qty/jam
- auto calculate subtotal

====================================================
MODUL BONUS TARGET
====================================================

Bonus target dihitung dari TOTAL PENJUALAN GABUNGAN semua cabang.

Contoh:
WA = 50.000.000
Marketplace = 300.000.000
Web = 10.000.000

Total = 360.000.000

====================================================
MASTER BONUS TARGET
====================================================

HRD dapat membuat multiple target.

Contoh:
- >100JT = 200.000
- >200JT = 300.000
- >300JT = 400.000

RULE:
- sistem mengambil target tertinggi yang tercapai
- bukan akumulasi

====================================================
PEMBAGIAN BONUS TARGET
====================================================

Bonus target dibagi rata ke semua karyawan aktif.

Contoh:
Bonus = 400.000
Jumlah Karyawan Aktif = 8

Maka:
400.000 / 8 = 50.000 per karyawan
