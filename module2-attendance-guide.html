02
Business Rules & Logic
🔒
1x Check-in per Hari
Database constraint UNIQUE(user_id, date). Jika sudah check-in dan coba lagi, API return 409 Conflict dengan pesan yang jelas.
HTTP 409
⏰
Auto-Status Terlambat
Server WIB (UTC+7) menentukan status. Check-in sebelum 08:05 → present. Check-in setelah 08:05 → late. Toleransi 5 menit dari jam masuk 08:00.
LATE_GRACE = 5 mnt
📍
GPS Optional
Koordinat GPS tidak wajib — jika user menolak izin lokasi, absensi tetap berhasil dengan lat/lng = NULL. Jika ada, disimpan ke check_in_lat & check_in_lng.
Non-blocking
🧮
Auto Hitung Jam Kerja
Saat check-out, sistem otomatis menghitung selisih waktu check_in → check_out dalam jam (2 desimal) dan menyimpan ke kolom work_hours.
work_hours = out − in
🕐
Waktu Server WIB
Semua waktu dihitung di sisi server menggunakan WIB (UTC+7), bukan waktu client. Ini mencegah manipulasi waktu dari device user.
Server-side time
🔄
Urutan Wajib
Check-out hanya bisa dilakukan setelah check-in di hari yang sama. Tidak bisa check-out tanpa check-in. Tidak bisa check-out 2x di hari yang sama.
HTTP 400 / 409
03
Check-in Logic Flow
1
User klik tombol CHECK IN
Frontend mengambil GPS (non-blocking, timeout 10 detik). Jika GPS gagal/ditolak, lanjut tanpa koordinat.
GPS optional
2
POST /api/attendance/check-in
Kirim payload { lat, lng } ke backend. JWT diperlukan di header Authorization.
3
Backend cek duplikat
Query WHERE user_id = ? AND date = today_WIB. Jika sudah ada record → return 409.
ALREADY_CHECKED_IN → 409
4
Hitung status berdasarkan jam server WIB
Ambil waktu WIB server → konversi ke HH:MM → bandingkan dengan threshold 08:05.
≤ 08:05 → present  > 08:05 → late
5
INSERT ke tabel attendance
Simpan: user_id, date, check_in (HH:MM:SS), check_in_lat, check_in_lng, status.
201 Created
6
Response ke frontend
Return data attendance baru + pesan sesuai status (tepat waktu / terlambat). Frontend refresh tampilan dan update tombol ke CHECK OUT.

Cara Test Module 2
ℹ️
Ganti TOKEN dengan accessToken yang didapat dari login endpoint POST /api/auth/login
BASH1. Cek status hari ini
curl http://localhost:5000/api/attendance/today \
  -H "Authorization: Bearer TOKEN"
BASH2. Check-in (dengan GPS)
curl -X POST http://localhost:5000/api/attendance/check-in \
  -H "Authorization: Bearer TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"lat":-6.2088,"lng":106.8456}'
BASH3. Check-out
curl -X POST http://localhost:5000/api/attendance/check-out \
  -H "Authorization: Bearer TOKEN" \
  -H "Content-Type: application/json" \
  -d '{}'
BASH4. Riwayat bulan ini
curl "http://localhost:5000/api/attendance/history?month=2024-01" \
  -H "Authorization: Bearer TOKEN"
BASH5. Admin — rekap harian semua karyawan
curl "http://localhost:5000/api/attendance/admin/daily?date=2024-01-15" \
  -H "Authorization: Bearer ADMIN_TOKEN"
06
UI Components Dibangun
⭕ Big Action Button
Tombol bulat 160×160px dengan pulse ring animation. 3 state: Check-in (hijau/amber), Check-out (biru), Selesai (grey). Warna berubah jika sudah melewati jam 08:05.
🕐 Live Clock
Digital clock WIB update tiap detik via useLiveClock hook. Tampil jam:menit:detik + hari + tanggal Indonesia. Banner peringatan muncul jika sudah melewati 08:05.
📍 GPS Indicator
Auto-request GPS saat halaman dibuka. Tampil koordinat + akurasi jika berhasil, atau "Tanpa GPS" jika ditolak. Tombol Refresh GPS tersedia. Absensi tetap jalan tanpa GPS.
📊 Today Summary Card
Card ringkasan berisi: jam masuk, jam keluar, total jam kerja. Status badge berwarna (Hadir/Terlambat). Koordinat GPS tampil jika tersedia.
📅 History Tab
Filter per bulan dengan navigasi prev/next. Stats 4 kolom (hari, hadir, telat, jam). List per record dengan tanggal, waktu in/out, duration, GPS indicator, dan status badge.
📈 Dashboard Quick Card
Dashboard card absensi sekarang live — menampilkan status hari ini (Belum Absen / Sudah Masuk / Lengkap) dengan warna dinamis dan jam check-in/out.
