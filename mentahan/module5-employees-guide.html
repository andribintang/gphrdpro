04
UI Components — EmployeesPage.jsx
📊 StatsRow (4 cards)
Total · Aktif · Tidak Aktif · Baru Bulan Ini. Semua data live dari GET /employees/stats. Hanya tampil untuk admin/HR/supervisor.
🔍 Search + Filter Panel
Search bar dengan debounce 350ms. Filter toggle button berubah biru jika aktif. Filter panel slide-down: status (semua/aktif/berhenti) dan departemen dari data aktual.
👥 Employee List
Card per baris: avatar warna deterministik dari nama, nama + role badge + status badge + masa kerja. Klik buka ProfileDrawer.
📋 ProfileDrawer
Bottom-sheet detail: avatar besar, stats 3-kolom (hadir/cuti/gaji), info kerja lengkap, kontak darurat. Tombol Edit + Nonaktifkan untuk admin/HR.
✍ EmployeeForm (2-Step)
Step 1: nama, email, password, NIP, role, jabatan, dept, telepon. Step 2: gaji, tanggal bergabung, status, alamat, kontak darurat. Progress indicator 2 dots.
⚠ Deactivate Confirm Modal
Konfirmasi center-modal sebelum deactivate. Menjelaskan bahwa akun tidak bisa login namun data tetap tersimpan (soft delete).

Test Commands
BASHTambah karyawan baru
curl -X POST http://localhost:5000/api/employees \
  -H "Authorization: Bearer HR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Rina Kusuma",
    "email": "rina@company.com",
    "password": "Rina@12345",
    "nip": "NIP-005",
    "position": "Staff Marketing",
    "department": "Marketing",
    "salary_base": 4500000,
    "join_date": "2024-02-01",
    "role": "employee"
  }'
BASHCari karyawan + filter departemen
curl "http://localhost:5000/api/employees?search=ahmad&department=Technology" \
  -H "Authorization: Bearer HR_TOKEN"
BASHProfil karyawan lengkap (ID=3)
curl http://localhost:5000/api/employees/3 \
  -H "Authorization: Bearer TOKEN"
BASHNonaktifkan karyawan
curl -X PATCH http://localhost:5000/api/employees/5/deactivate \
  -H "Authorization: Bearer HR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"status":"terminated"}'
BASHStats karyawan
curl http://localhost:5000/api/employees/stats \
  -H "Authorization: Bearer HR_TOKEN"
