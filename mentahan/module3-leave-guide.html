02
Approval Workflow Flow
1
Karyawan ajukan cuti via bottom-sheet form
Pilih tipe, tanggal, alasan (min 10 karakter). Preview hari kerja dihitung real-time di frontend (exclude Sabtu/Minggu). Warning jika melebihi kuota.
Client-side preview
2
POST /api/leaves — validasi 3 lapis di backend
Server cek: (a) tanggal tidak di masa lalu, (b) tidak overlap dengan cuti pending/approved lain, (c) untuk annual: kuota mencukupi. Hari kerja dihitung ulang di server.
Past date → 400Overlap → 409Quota habis → 400
3
Record tersimpan status "pending"
LeaveRequest dibuat. Karyawan lihat badge amber "Menunggu" di tab "Cuti Saya". Tombol Batalkan tersedia selama masih pending.
201 Created
4
HR/Supervisor buka tab "Persetujuan"
Kartu menampilkan nama, departemen, tipe cuti, tanggal, jumlah hari, alasan. Dua tombol: Setujui dan Tolak.
Role: admin / hr / supervisor
5A
Setujui → PATCH /:id/approve
Status → approved. Annual: annual_used += total_days. Sick: sick_used += total_days. Semua dalam DB transaction agar atomic.
Quota dikurangi otomatis · Transaction safe
5B
Tolak → RejectModal muncul (bukan browser prompt)
Bottom-sheet modal meminta alasan penolakan (opsional). Klik "Tolak Cuti" → PATCH /:id/reject. Status → rejected, alasan tersimpan. Quota TIDAK berubah.
Quota tidak dikurangi
6
Karyawan lihat hasil di "Cuti Saya"
Badge berubah: hijau "Disetujui" atau merah "Ditolak". Alasan penolakan tampil dalam card. QuotaCard progress bar ter-update.
Real-time refresh setelah aksi

Test Commands
ⓘ
Ganti TOKEN dengan accessToken dari POST /api/auth/login. Jalankan node scripts/migrate.js untuk tabel leave_quotas + seed data.
BASHCek kuota cuti
curl "http://localhost:5000/api/leaves/quota?year=2024" \
  -H "Authorization: Bearer EMP_TOKEN"
BASHAjukan cuti tahunan
curl -X POST http://localhost:5000/api/leaves \
  -H "Authorization: Bearer EMP_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"type":"annual","start_date":"2024-02-05","end_date":"2024-02-07","reason":"Liburan keluarga ke Yogyakarta"}'
BASHHR: setujui cuti ID=1
curl -X PATCH http://localhost:5000/api/leaves/1/approve \
  -H "Authorization: Bearer HR_TOKEN"
BASHHR: tolak dengan alasan
curl -X PATCH http://localhost:5000/api/leaves/2/reject \
  -H "Authorization: Bearer HR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"rejection_reason":"Jadwal bentrok dengan project deadline"}'

UI Components
📊 QuotaCard
Progress bar visual berubah warna: hijau → amber (50%+) → merah (80%+). Stats: Sisa Tahunan, Carry Over, Sakit Dipakai.
📝 ApplyForm (Bottom Sheet)
Slide-up modal. Grid 2-kolom 7 tipe cuti. Date picker. Live preview hari kerja. Warning quota real-time. Textarea + counter karakter.
🃏 LeaveCard
Color stripe sesuai status. Icon tipe. Date range. Alasan. Rejection reason. Tombol Batalkan atau Setujui+Tolak sesuai role.
❌ RejectModal
Menggantikan browser prompt(). Bottom-sheet mobile-friendly dengan textarea alasan dan tombol konfirmasi merah.
👤 Tab: Cuti Saya
Filter pills (Semua/Menunggu/Disetujui/Ditolak). Refresh button. Skeleton loader. QuotaCard + tombol Ajukan Cuti.
👥 Tab: Persetujuan
Hanya muncul untuk admin/hr/supervisor. Stats pending count. Toggle pending/semua. Tiap card punya aksi approve+reject.
