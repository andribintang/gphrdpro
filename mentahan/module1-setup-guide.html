02
ENV Files
ENV backend/.env
# ── Server ──────────────────────────────────
PORT=5000
NODE_ENV=development

# ── Railway MySQL ────────────────────────────
DB_HOST=containers-us-west-xxx.railway.app
DB_PORT=7890
DB_NAME=hrd_system
DB_USER=root
DB_PASSWORD=your_railway_password

# ── JWT ─────────────────────────────────────
JWT_SECRET=ganti_ini_minimal_32_karakter_random_string
JWT_EXPIRES_IN=7d
JWT_REFRESH_EXPIRES_IN=30d

# ── CORS ────────────────────────────────────
FRONTEND_URL=http://localhost:5173
ENV frontend/.env
VITE_API_URL=http://localhost:5000/api
⚠️
Railway MySQL: Buka dashboard Railway → klik service MySQL → tab Connect → copy Host, Port, Database, Username, Password ke .env backend. Pastikan MySQL service sudah aktif sebelum menjalankan migrate.
03
Install Commands
BASH Backend — install & setup
# 1. Masuk ke folder backend
cd hrd-system/backend

# 2. Install semua dependencies
npm install

# 3. Buat file .env dari template
cp .env.example .env
# → Edit .env, isi Railway MySQL credentials

# 4. Jalankan migrasi + seed data
node scripts/migrate.js

# 5. Start development server
npm run dev
# ✅ API running → http://localhost:5000
BASH Frontend — install & run
# 1. Masuk ke folder frontend
cd hrd-system/frontend

# 2. Install semua dependencies
npm install

# 3. Buat file .env
cp .env.example .env

# 4. Start development server
npm run dev
# ✅ App running → http://localhost:5173

06
Running Instructions
1
Buat project di Railway
Login ke railway.app → New Project → Add MySQL plugin → copy credentials dari tab Connect ke backend .env
2
Install & migrate backend
cd backend && npm install && node scripts/migrate.js — akan membuat 5 tabel dan 4 akun demo secara otomatis
3
Start backend dev server
npm run dev → server aktif di http://localhost:5000 dengan hot-reload via nodemon
4
Install & start frontend
cd frontend && npm install && npm run dev → app buka di http://localhost:5173
5
Test login via UI
Buka browser → klik salah satu tombol demo role → klik Masuk → lihat dashboard dengan bottom nav mobile
6
Verifikasi API via curl / Postman
Test endpoint GET /health dan POST /api/auth/login untuk konfirmasi backend OK
07
Test Commands (cURL)
BASH Health check
curl http://localhost:5000/health
BASH Login sebagai Admin
curl -X POST http://localhost:5000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@hrd.com","password":"Admin@123"}'
BASH Get profil (gunakan token dari login)
curl http://localhost:5000/api/auth/me \
  -H "Authorization: Bearer <ACCESS_TOKEN>"
