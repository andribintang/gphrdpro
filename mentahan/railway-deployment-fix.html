502 Bad Gateway — Root Cause Analysis

HRD Lite Railway
Deployment Fix

Error 502 di Railway selalu disebabkan oleh server yang gagal start atau tidak bisa di-reach. Ditemukan 4 bug kritis yang menyebabkan container Railway crash sebelum bisa menerima request.

⚠ process.exit(1) saat DB fail ⚠ app.listen tanpa 0.0.0.0 ⚠ DATABASE_URL tidak didukung ⚠ CORS blokir Railway domain
!
4 Root Cause dan Fix
❌ SEBELUM (Bermasalah)
✅ SETELAH (Fixed)
Bug 1: process.exit(1) saat DB gagal → Railway kill container → 502
Retry 5x dengan jeda 3 detik, lalu server tetap berjalan meski DB belum siap
Bug 2: app.listen(PORT) → Railway tidak bisa route traffic
app.listen(PORT, '0.0.0.0') → bind ke semua interface
Bug 3: Hanya support DB_HOST, DB_NAME... → Railway pakai DATABASE_URL
Support keduanya: cek DATABASE_URL dulu, fallback ke individual vars
Bug 4: CORS hanya allow localhost → Railway frontend domain diblokir
CORS whitelist gphrdpro-production.up.railway.app + dynamic via FRONTEND_URL
01
Upload File yang Sudah Diperbaiki

Download ZIP dari chat, extract, lalu replace 2 file berikut di Railway project kamu:

FILESFile yang harus di-replace
backend/config/database.js   ← Support DATABASE_URL + retry logic, no process.exit
backend/server.js             ← bind 0.0.0.0, CORS fix, DB tidak block startup
backend/package.json          ← tambah engines: { node: ">=18.0.0" }
backend/railway.json          ← Railway deploy config (BARU)
backend/Procfile              ← Fallback start command (BARU)
02
Set Environment Variables di Railway
⚠️
Ini langkah yang paling sering terlewat. Buka Railway Dashboard → pilih service backend → tab Variables → pastikan semua variable di bawah ini ada.
ENVVariables yang WAJIB ada di Railway Dashboard
# ── Wajib diisi manual ────────────────────────────────
NODE_ENV=production
JWT_SECRET=isi_string_random_panjang_min_32_karakter_disini
JWT_EXPIRES_IN=7d
JWT_REFRESH_EXPIRES_IN=30d
FRONTEND_URL=https://gphrdpro-production.up.railway.app

# ── Railway auto-set (tidak perlu diisi manual) ───────
# DATABASE_URL  → auto-set jika MySQL service terhubung
# PORT          → auto-set oleh Railway (jangan diisi manual!)
⚠️
JANGAN set PORT secara manual di Railway Variables. Railway auto-set PORT dan server harus menggunakan nilai tersebut. Jika PORT di-hardcode, container tidak bisa di-reach.
CHECKCara verifikasi DATABASE_URL sudah terhubung
1. Railway Dashboard → pilih service backend
2. Tab Variables
3. Cari variable DATABASE_URL
4. Jika tidak ada → berarti MySQL service belum di-link ke backend
5. Cara link: Railway Dashboard → backend service → Settings →
   Service Connections → tambahkan MySQL service → Save
6. Setelah link, Railway otomatis inject DATABASE_URL
